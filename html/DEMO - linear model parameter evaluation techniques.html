Hypothesis testing of intercept and slope point estimators¶

hypothesis test for slope parameter:                    hypothesis test for intercept parameter:¶

${H_0}: {B_1}=0$                                                            ${H_0}: {B_0}=B_{0,0}$¶

${H_1}: {B_1}\ne0$                                                            ${H_1}: {B_0}\ne{B_{0,0}}$¶

if  ${|t_0|} > {t_{\alpha/2}}$, then we can reject the null hypothesis.¶

note: the critical value ${t{\alpha/2}}$ is determined from the t-distribution with n-2 degrees of freedom_

F-test for significance of linear relationship¶

This tests that each coefficient is jointly statistically significantly different from zero. In layman's terms...

Null hypothesis: The fit of the intercept-only model and the current model are equal.
Alternative hypothesis: The fit of the intercept-only model is significantly reduced compared to the current model.

if  ${|F_0|} > {F_{\alpha;1;175}}$, then we can reject the null hypothesis.¶

Conclusion:¶

From these figures, it is clear that this linear model is a poor fit.
1) The probability scale residuals diverge considerably from a tight linear trend.
2) The boxplot, though symmetrical, has many outliers.
3) The histogram shows a non-symmetrical, non-normal distribution.

These three graphs bring into question the normality of the residuals (an important assumption for OLS) and seem to indicate that errors are not independently distributed.
If the errors are heteroskedastic they will require the use of a weighted least squares approach.
